# Project Bot - Claude Code 규칙

이 프로젝트에는 Project Bot MCP 서버가 등록되어 있습니다.
아래 규칙을 반드시 따라주세요.

## 핵심 규칙: 매 턴 send_notification 호출

**모든 턴이 끝날 때 반드시 `send_notification` 도구를 호출하세요.**

이 규칙은 팀원이 Discord에서 작업 진행 상황을 실시간으로 확인하기 위한 것입니다.

## event_type 선택 가이드

| 상황 | event_type | 예시 메시지 |
|------|-----------|------------|
| 계획을 세울 때 | `plan` | "로그인 기능 구현 계획 수립" |
| 질문을 제시할 때 | `question` | "인증 방식을 JWT vs 세션 중 어떤 것으로 할까요?" |
| 작업을 완료했을 때 | `complete` | "로그인 API 엔드포인트 구현 완료" |
| 에러가 발생했을 때 | `error` | "테스트 실패: test_login_validation" |
| 빌드를 실행할 때 | `build` | "프로젝트 빌드 진행 중" |
| 테스트를 실행할 때 | `test` | "단위 테스트 42개 실행 중" |
| 배포를 진행할 때 | `deploy` | "스테이징 환경에 배포 시작" |

## 호출 예시

```
send_notification(
    project_name="my-app",
    message="사용자 인증 모듈 구현 완료. JWT 토큰 발급 및 검증 로직 추가",
    event_type="complete"
)
```

## 메시지 작성 가이드

- 한 줄로 간결하게 작성
- 무엇을 했는지 (또는 하고 있는지) 명확히 표현
- 기술적 세부사항보다는 작업의 의미를 전달

## 주의사항

- 도구 호출에 실패해도 작업 자체는 계속 진행하세요
- project_name은 현재 작업 중인 프로젝트명을 사용하세요
- 한 턴에 여러 작업을 했다면 가장 중요한 작업 하나로 요약하세요
